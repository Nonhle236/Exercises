SQL fundamentals

Exercises 1 through to 4 highlight the use of SQL fundamentals with proper analysis of data and understanding. The proper use and understanding of a data warehouse, databases and how we retrieve data from them. Handwritten without the use of a tool with an aim to understand the SQL language.

# Excercise 1
This exercise mainly focuses on SELECT statements. How you can distinctly identify unique values in a column using a SELECT DISTINCT statement. How to order your rows in ascending or descending order and LIMIT the number of rows you want to analyse. The use of logical operaters WHERE you can specify conditions to filter your data. (AND, OR, BBETWEEN and NOT).

# Exercise 2
This exercise focuses on functions that summarise multiple rows into one result. Aggregate functions operate row by row on a specific column and return a single value. COUNT() counts the uniquily identified values, SUM() gives the total of a numeric column, AVG() calculates the average of a numeric column, and MAX() and MIN() individually give the largest/highest value and the smallest/lowest value of a numeric column. Combining the functions together with SELECT statemntfoundations from exercise 1.
